Modules

When building a complex interactive SVG animation eventually you will might end up with a substantially amount of code. Structuring your code can help you keeping things clear. In this project we will work with ES Modules. You need to run your code in a webserver for the modules to work! I use the http-server package for this, but you can use your own preferred webserver.

Step 2: Preparing your SVG 20 mins

Add SVG code to html

Open your SVG file in your text editor and copy / paste the code into the index.html file.

If you did not bring a SVG file. Pick an awesome free vector graphic (EPS) from Freepik. Open the EPS file in a vector programme like Illustrator and export it to SVG... or convert the file online @ eEPS to SVG.. or draw your own vector graphic while you are at it!

Make sure the code is as clean as possible, and give a class or id attribute to the elements you want to animate. Also optimize your svg with the more then spledid tool from Jake Archibald: SVGOMG.

Step 3: Animating with GreenSock 60 mins

Tell the story with code

  1. Make a chapter module for each chapters of your storyboard
  2. Create a main timeline in each chapter module
  3. Create timelines for every actor which has a role in the specific chapter module
  4. Add the child timelines to the main timeline in order of appearance
  5. Tween the (different parts of the) actors until you are done with the chapter
  6. Add some buttons for controlling which chapter play when.
